# <a name="use-the-new-authentication-for-your-storsimple"></a>StorSimple에 새 인증 사용

## <a name="overview"></a>개요

StorSimple 장치 관리자 서비스는 Microsoft Azure에서 실행되며 여러 StorSimple 가상 배열에 연결됩니다. 현재까지 StorSimple 장치 관리자 서비스는 ACS(Access Control Service)를 사용하여 StorSimple 장치에 대한 서비스를 인증했습니다. ACS 메커니즘은 조만간 사용되지 않고 AAD(Azure Active Directory) 인증으로 대체될 예정입니다.

이 문서에 포함된 정보는 StorSimple 1200 시리즈 Virtual Array이에만 적용됩니다. 이 문서에서는 StorSimple 장치에 적용할 수 있는 방화벽 규칙에 대한 수정과 AAD 인증 및 연결된 신규 서비스 등록 키에 대해 자세히 설명합니다.

AAD 인증은 업데이트 1 이상을 실행하는 StorSimple Virtual Array(모델 1200)에서 발생합니다.

AAD 인증 도입으로 인해 다음과 항목이 변경됩니다.

- 방화벽 규칙에 대한 URL 패턴
- 서비스 등록 키

이러한 변경 사항은 다음 섹션에서 자세히 설명합니다.

## <a name="url-changes-for-aad-authentication"></a>AAD 인증에 대한 URL 변경

서비스에서 AAD 기반 인증을 사용하도록 보장하려면 모든 사용자가 새 인증 URL을 방화벽 규칙에 포함시켜야 합니다.

StorSimple Virtual Array를 사용하는 경우에는 방화벽 규칙에 다음 URL이 포함되어 있는지 확인합니다.

| URL 패턴                         | 클라우드 | 구성 요소/기능         |
|------------------------------------|-------|---------------------------------|
| `https://login.windows.net`        | Azure 공용 |AAD 인증 서비스      |
| `https://login.microsoftonline.us` | 미국 정부 |AAD 인증 서비스      |

StorSimple Virtual Array에 대한 URL 패턴의 전체 목록은 [방화벽 규칙에 대한 URL 패턴](storsimple-ova-system-requirements.md#url-patterns-for-firewall-rules)을 참조하세요.

지원 중단 날짜가 지나도 인증 URL이 방화벽 규칙에 포함되지 않은 경우 StorSimple 장치가 서비스에서 인증을 받을 수 없다는 중요한 경고가 표시됩니다. 이 서비스는 장치와 통신할 수 없습니다. 이 경고가 표시되면 새 인증 URL을 포함해야 합니다. 경고에 대한 자세한 내용은 [경고를 사용하여 StorSimple 장치 모니터링](storsimple-virtual-array-manage-alerts.md#networking-alerts)을 참조하세요.

## <a name="device-version-and-authentication-changes"></a>장치 버전 및 인증 변경

StorSimple Virtual Array를 사용하는 경우 다음 테이블을 사용하여 실행 중인 장치 소프트웨어 버전을 기반으로 수행할 작업을 결정합니다.

| 실행 중인 장치  | 수행할 작업                                    |
|----------------------------|--------------------------------------------------------------|
| 업데이트 1.0 이상을 실행 중이고 오프라인 상태입니다. <br> URL이 허용 목록에 없다는 경고가 표시됩니다.| 인증 URL을 포함하도록 방화벽 규칙을 수정합니다. [인증 URL](#url-changes-for-aad-authentication)을 참조하세요. |
| 업데이트 1.0 이상을 실행 중이고 온라인 상태입니다.| 별도의 작업이 필요하지 않습니다.                                       |
| 업데이트 0.6 이상을 실행 중이고 장치가 오프라인 상태입니다. | [카탈로그 서버를 통해 업데이트 1.0을 다운로드합니다](storsimple-virtual-array-install-update-1.md#download-the-update-or-the-hotfix).<br>[로컬 웹 UI를 통해 업데이트 1.0을 적용합니다](storsimple-virtual-array-install-update-1.md#install-the-update-or-the-hotfix). <br> [서비스에서 AAD 등록 키를 받습니다](#aad-based-registration-keys). <br> 1-5단계를 수행하여 [가상 배열의 Windows PowerShell 인터페이스에 연결](storsimple-virtual-array-deploy2-provision-hyperv.md#step-2-provision-a-virtual-array-in-hypervisor)합니다.<br> `Invoke-HcsReRegister` cmdlet을 사용하여 Windows PowerShell을 통해 장치를 등록합니다. 이전 단계에서 얻은 키를 제공합니다.|
| 업데이트 0.6 이상을 실행 중이고 장치가 온라인 상태입니다. | 인증 URL을 포함하도록 방화벽 규칙을 수정합니다.<br> Azure Portal을 통해 업데이트 1.0을 설치합니다. |

## <a name="aad-based-registration-keys"></a>AAD 기반 등록 키

StorSimple Virtual Array에 대한 업데이트 1.0부터는 새로운 AAD 기반 등록 키가 사용됩니다. 이 등록 키를 사용하여 StorSimple 장치 관리자 서비스를 장치에 등록합니다.

Update 0.6 이전 버전을 실행하는 StorSimple Virtual Array를 사용하는 경우 새 AAD 서비스 등록 키를 사용할 수 없습니다. 서비스 등록 키를 다시 생성해야 합니다. 키를 다시 생성하면 모든 후속 장치를 등록하는 데 새 키가 사용됩니다. 이전 키는 더 이상 유효하지 않습니다.

- 새 AAD 등록 키는 3일 후에 만료됩니다.
- AAD 등록 키는 업데이트 1 이상을 실행하는 StorSimple 1200 Virtual Array에서만 작동합니다. StorSimple 8000 시리즈 장치의 AAD 등록 키는 작동하지 않습니다.
- AAD 등록 키는 해당 ACS 등록 키보다 깁니다.

AAD 서비스 등록 키를 생성하려면 다음 단계를 수행합니다.

#### <a name="to-generate-the-aad-service-registration-key"></a>AAD 서비스 등록 키를 생성하려면

1. **StorSimple 장치 관리자**에서 **관리 &gt;** **키**로 이동합니다.
    
    ![키로 이동](./media/storsimple-virtual-array-aad-registration-key/aad-registration-key1.png)

2. **키 생성**을 클릭합니다.

    ![다시 생성 클릭](./media/storsimple-virtual-array-aad-registration-key/aad-click-generate-registration-key.png)

3. 새 키를 복사합니다. 이전 키는 더 이상 유효하지 않습니다.

    ![다시 생성 확인](./media/storsimple-virtual-array-aad-registration-key/aad-registration-key2.png)
